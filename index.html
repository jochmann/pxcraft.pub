development guide (alpha)

UI-logic

layout.css applies styles to semantic elements. No classes.

helper.css goes with helper.js and is an optional add on to programmatically enhance vanilla HTML with further layout niceties and functionality.

The user interface is sectioned off in the document (commented in HTML). It can be added after the footer. With the user interface go ui.css and ui.js to programmatically enhance vanilla HTML5 (book-)documents.

is-modal & is-active classes used to apply dynamic styles to switch the UI elements on/off

imprint & toc & comments are IDs relevant to meta-information about the document. They are (partly) populated by scripts from other parts of the document and turned into modals programmatically on load.

We test browser capabilities (with modernizr customized javascript) to warn users who don't support important features. Test script is called in helper.js

Classes in use to toggle modals: js-toggle-imprint, js-toggle-toc, js-toggle-?...

naming-conventions

sorry, not fully streamlined yet

css is interlinked (no camel case) at molecular segments. Look if a segment could work as a component of another class (or id).

e.g.
class="ui-menu" class="ui-menu-button" class="menu-element-button-wrapper"

established names
is- prefix for dynamically applied classes
js- prefix for classes on which javascript relies to trigger events

verbs (concatenated) before noun they apply to. e.g. js-scrollto-footer
